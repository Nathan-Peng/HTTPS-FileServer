HTTPS-FileServer
 
纯Python标准库开发 · 轻量化局域网HTTPS加密文件服务
零外部二进制依赖、自动生成SSL证书、全页面内嵌无需外部HTML文件，开箱即用内网文件共享工具。
适用于手机电脑互传文件、内网静态资源托管、个人轻量私有网盘、开发测试文件分发场景。
 
✨ 核心亮点
 
🔐 安全特性
 
1. 全程HTTPS加密传输，仅开放8443加密端口，无明文HTTP服务，防止内网抓包泄露文件
2. Python原生 cryptography 生成自签名证书，无需手动安装OpenSSL程序，跨平台通用
3. 权限隔离：本机 127.x.x.x 回环IP仅只读浏览，禁止上传/删除；仅局域网设备拥有修改权限
4. 双层上传防护：可执行文件后缀黑名单 + 文件二进制魔数校验，拦截修改后缀伪装的病毒文件
5. 路径严格校验防护目录穿越攻击，无法访问项目根目录以外系统文件
6. 禁用老旧不安全TLS 1.0/1.1协议，采用高强度加密套件加固传输安全

🚀 快速部署教程
 
环境依赖
 
- Python 3.8 及以上版本
- 内置标准库全部自带，仅证书生成依赖第三方包  cryptography 
 
步骤1：克隆仓库 & 安装依赖
 
```bash
git clone https://github.com/你的用户名/HTTPS-FileServer.git
cd HTTPS-FileServer
pip install cryptography
```
 
步骤2：一键启动服务
 
```bash
python main.py
```
 
✅ 首次运行自动检测证书缺失，后台自动生成有效期365天自签名SSL证书，无需手动操作
 
步骤3：局域网访问使用
 
1. 本机本地访问： https://127.0.0.1:8443 
2. 同WiFi/局域网其他设备： https://你的局域网IP:8443 
 
浏览器提示「连接不安全」为自签名证书正常现象，点击 高级 → 继续访问 即可正常使用。

🛡️ 详细安全策略
 
1. 传输安全
 
- 仅TLS加密通信，彻底关闭明文HTTP端口，内网无法抓包窃取文件内容
- 淘汰老旧弱加密协议，仅保留现代高强度加密套件
 
2. 上传安全限制
 
- 禁止后缀： .exe/.bat/.ps1/.dll/.sys/.cmd/.com  等全部可执行程序
- 二进制魔数校验：读取文件头部字节判断真实类型，修改文件后缀伪装病毒直接拦截
- 单文件最大上传体积：10MB，防止超大文件占用服务器资源
 
3. 权限分级
 
- 本地127网段IP：仅文件浏览只读权限，禁止上传、删除、打包修改操作
- 局域网设备IP：拥有完整上传、删除、目录打包全部操作权限
 
📊 日志系统说明
 
1. 双路输出：控制台实时打印日志 + 本地 server.log 文件持久化保存
2. 内置字符清洗算法，自动过滤乱码控制字符、非法二进制数据，日志整洁可读
3. 标准化日志格式： [时间戳] IP:端口 | 请求方法 路径 状态码 
4. 异常断开、网络波动优雅处理，无冗余报错刷屏

# HTTPS-FileServer
轻量级纯Python局域网HTTPS文件服务，无第三方复杂依赖，开箱即用。
仅开放加密8443端口，杜绝明文HTTP风险；支持文件浏览、拖拽上传、目录打包、文件删除、实时监控面板，自动深色模式适配。

## ✨ 核心功能
1. 全HTTPS加密传输，仅8443端口，无明文HTTP访问，规避SSL协议报错弹窗
2. 文件管理器：根目录/上传目录浏览，文件大小、修改时间、图标分类展示
3. 拖拽多文件上传：仅局域网设备可上传，本机127.0.0.1禁止上传/删除（安全限制）
4. 自动安全校验：文件头部魔数校验，拦截修改后缀伪装的可执行病毒文件
5. 目录一键ZIP打包：提供可视化打包页面，支持根目录/上传文件夹快速下载压缩包
6. 文件删除接口：仅局域网IP可删除上传文件
7. 实时监控面板：展示运行时长、活跃连接、上传文件总数、磁盘占用、总请求统计
8. 自动分日期归档上传文件：`uploaded/20260629/xxx.jpg` 按天分类，便于整理
9. 日志净化过滤：自动屏蔽二进制乱码控制字符，日志文件干净可读
10. Gzip传输压缩 + 静态资源长缓存，访问速度更快
11. 自适应明暗主题：跟随系统自动切换深色/浅色界面

## 📦 环境依赖
仅Python标准库，**无需pip安装第三方包**
- Python 3.8+ 全部内置依赖：ssl/json/zipfile/threading等

## 🚀 Windows快速启动教程
### 步骤1：克隆仓库
```bash
git clone https://github.com/你的用户名/HTTPS-FileServer.git
cd HTTPS-FileServer
 
 
步骤2：生成SSL加密证书（仅首次运行需要）
 
双击  generate_cert.bat  自动生成  server.crt  /  server.key  证书文件
 
无需填写任何信息，一键自签名证书
 
步骤3：启动服务
 
bash
  
python main.py
 
 
步骤4：局域网访问
 
1. 本机浏览器： https://127.0.0.1:8443 
2. 手机/其他电脑（同WiFi）： https://192.168.xx.xx:8443 
 
浏览器提示「不安全」是自签名证书正常现象，点击「高级→继续访问」即可
 
📍 全部页面路由
 
路由地址 功能说明 
 /  根目录文件浏览 
 /upload.html  拖拽文件上传页面 
 /uploaded  上传文件归档目录（按日期分文件夹） 
 /dashboard  实时服务监控面板 
 /zip.html  可视化目录打包ZIP工具 
 /api/status  服务状态JSON接口 
 /zip?path=xxx  后台打包接口 
 
⚠️ 安全设计说明
 
1. 仅HTTPS 8443端口，无明文HTTP 80端口，避免抓包泄露文件
2. 本机回环地址  127.0.0.1  禁止上传/删除文件，仅局域网设备可操作
3. 拦截exe/bat/dll等可执行文件上传；文件头部魔数校验，防止伪装后缀病毒
4. 目录穿越防护：限制仅能访问程序运行根目录内文件

- 日志自动写入  server.log ，控制台同步输出
- 自动过滤二进制不可见乱码字符（如  ），统一替换为占位符 �
